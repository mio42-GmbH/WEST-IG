Element Operationalisierungshinweis

Übergreifend

Eine Historisierung der Daten ist mit dem Element Ressource.meta abzubilden.

meta.versionId: Angabe der fortlaufend aufsteigenden Versionsnummer der Historisierten Daten. Bei nicht historisierten Daten wird hier immer der Wert=1 verwendet.
meta.lastUpdated: Angabe des aktuellen Datums der Erstellung der Ressource

ARBEITSPAKET-2 / Medikations-Information / Verabreichung/Einnahme: Zeitangabe-Auswahl

Für erstellende und anzeigende Systeme:

  • Ist dieses Element nicht befüllt oder existiert ein Zeitraum ohne Enddatum, so sollte die Medikation als Dauermedikation gekennzeichnet werden.

ARBEITSPAKET-2 / Medikations-Information / Dosierung
Für erstellende Systeme:
  • Um eine eindeutige Abbildung von Dosierungen zu gewährleisten sollte jede Verabreichung/Einnahme und dessen Wiederholung als eigenständige Dosierungsangabe (Element Dosage in FHIR) dargestellt werden.
  • Es wird empfohlen, grundsätzlich zwischen einer strukturierten Dosieranweisung und einer Freitext Dosieranweisung zu unterscheiden, die sich gegenseitig ausschließen sollten. Im Falle einer strukturierten Dosieranweisung sollte die Gruppe "Dosierung der einzelnen Verabreichung/Einnahme" grundsätzlich befüllt werden.
  • Innerhalb des FHIR®-Elements Dosage.timing.repeat werden innerhalb von FHIR® verschiedene Regeln definiert (siehe https://www.hl7.org/fhir/datatypes.html#Timing). Hinweise bzw. Warnmeldungen zur Einhaltung dieser Regeln sollten benutzerfreundlich in die GUI des Primärsystems integriert werden, damit eine syntaktisch korrekte Instanz der FHIR®-Ressource erzeugt werden kann.
  • Über diese Regeln hinaus sind in den einzelnen Elementen dieses Informationsmodells weitere Hinweise zur Umsetzung einer benutzerfreundlichen GUI aufgeführt.

ARBEITSPAKET-2 / Medikations-Information / Dosierung / Dosierung der einzelnen Verabreichung/Einnahme

Für erstellende Systeme:

  • Es sollten grundsätzlich entweder eine Menge pro Gabe/Einnahme oder ein Mengenbereich pro Gabe/Einnahme angegeben werden, damit die Dosierungsangabe schlüssig ist.
  • Bei einer Bedarfsmedikation (FHIR®: asNeededBoolean = true | asNeededCodeableConcept exisitert) kann zusätzlich zu einer definierten Menge bzw. einem Mengenbereich pro Gabe/Einnahme auch eine maximale Menge pro Gabe/Einnahme angegeben werden.

ARBEITSPAKET-2 / Medikations-Information / Dosierung / Wiederholung der Verabreichung/Einnahme / Zeitangabe-Auswahl (dosisspezifisch)
Für erstellende Systeme:
  • Der "Zeitangabe-Auswahl (dosisspezifisch)" kann für jede Dosierungsangabe (FHIR-Element Dosage) einzeln abgebildet werden.
  • Diese Angaben sollten nur erfolgen, wenn es mehrere "Dosage" Elemente gibt und sich die Zeiträume der Dosierungen unterscheiden.

ARBEITSPAKET-2 / Medikations-Information / Dosierung / Wiederholung der Verabreichung/Einnahme / Zeitangabe-Auswahl (dosisspezifisch) / Zeitraum (dosisspezifisch)

Für erstellende Systeme:

  • Wenn das Element "Dosierung" nur einmal vorkommt, kann das Element "Zeitraum (dosisspezifisch)" auf Basis der Angaben des Elements "Verabreichung/Einnahme: Zeitangabe-Auswahl" auf der obersten Ebene des Profils "Medikations-Information" automatisiert befüllt werden.

ARBEITSPAKET-2 / Medikations-Information / Dosierung / Wiederholung der Verabreichung/Einnahme / Uhrzeit

Für erstellende Systeme:

  • Es wird empfohlen, das Ausfüllen von Uhrzeiten in der Benutzeroberfläche ohne Sekundengenauigkeit zu gestalten.

ARBEITSPAKET-2 / Medikations-Information / Dosierung / Wiederholung der Verabreichung/Einnahme / Wochentag

Für erstellende Systeme:

  • Bei Angabe des Wochentags sollte als Einheit der Zeitspanne "Woche" verwendet werden.
  • Soll das Medikament an mehreren Tagen pro Woche verabreicht/eingenommen werden, so sollten hierfür getrennte Dosage-Elemente angelegt werden.
  • Soll das Medikament an sämtlichen Wochentagen verabreicht/eingenommen werden, so sollte als Einheit der Zeitspanne "Tag" verwendet und dieses Element nicht befüllt werden.

ARBEITSPAKET-2 / Diagnose / Codierung Code/Bezeichnung / Code-Auswahl / ICD-10-GM-Code / ICD-Diagnosesicherheit

Bei Angabe der ICD-Diagnosesicherheit ist die zusätzliche Angabe der äquivalenten Werte für clinicalStatus und verificationStatus zwingend. Die Äquivalente sind im HL7®-Leitfaden Basis DE ( https://ig.fhir.de/basisprofile-de/stable/Ressourcen-Condition-ICD-10-GM.html ) wie folgt festgelegt:

  • A (ausgeschlossen) => clinicalStatus = leer, verificationStatus="refuted"

  • G (gesicherte Diagnose) => clinicalStatus = "active", verificationStatus="confirmed"

  • V (Verdacht auf/zum Ausschluss von) => clinicalStatus = "active", verificationStatus="provisional" oder "differential"

  • Z (Zustand nach) => clinicalStatus = "resolved", verificationStatus="confirmed"
